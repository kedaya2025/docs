## 货币强度显示问题

### 强度值不更新

<AccordionGroup>
  <Accordion title="问题描述">
    货币强度卡片显示的强度值长时间不变化，或显示为0
  </Accordion>
  
  <Accordion title="可能原因">
    - 网络连接问题导致价格数据获取失败
    - CStrength类初始化失败
    - 相关货币对数据缺失或无效
    - 定时器更新机制故障
    - 系统时间设置错误
    - MT5数据源问题
  </Accordion>
  
  <Accordion title="解决方案">
    1. **检查网络连接**: 确认MT5平台网络连接正常
    2. **验证货币对数据**: 在MT5中检查相关货币对是否有数据
    3. **重启模块**: 关闭并重新打开篮子交易面板
    4. **检查系统时间**: 确保系统时间准确
    5. **重新连接服务器**: 重新连接MT5交易服务器
    6. **检查EA权限**: 确认EA有获取市场数据的权限
    7. **联系经纪商**: 如果问题持续，联系经纪商确认数据源
  </Accordion>
</AccordionGroup>

### 强度卡片显示异常

<AccordionGroup>
  <Accordion title="问题描述">
    强度卡片显示错乱、重叠或颜色异常
  </Accordion>
  
  <Accordion title="可能原因">
    - 位图资源生成失败
    - 内存不足导致资源损坏
    - 显卡驱动问题
    - 屏幕分辨率或DPI设置异常
    - CBmp位图引擎故障
  </Accordion>
  
  <Accordion title="解决方案">
    1. **重新生成位图**: 切换到其他模块再回来重新生成
    2. **释放内存**: 关闭其他不必要的程序释放内存
    3. **更新显卡驱动**: 确保显卡驱动为最新版本
    4. **调整分辨率**: 将屏幕分辨率设置为推荐值
    5. **重启MT5**: 重启MetaTrader 5平台
    6. **检查DPI设置**: 确认系统DPI设置为100%或125%
  </Accordion>
</AccordionGroup>

### 强度计算不准确

<AccordionGroup>
  <Accordion title="问题描述">
    强度值与实际市场情况不符，或各货币强度值异常
  </Accordion>
  
  <Accordion title="可能原因">
    - 货币对数据不完整
    - 强度计算算法参数错误
    - 基础数据时间不同步
    - 某些货币对流动性不足
    - 市场异常波动影响计算
  </Accordion>
  
  <Accordion title="解决方案">
    1. **检查数据完整性**: 确认所有相关货币对都有有效数据
    2. **同步数据时间**: 确保所有货币对数据时间一致
    3. **调整计算参数**: 根据市场情况调整强度计算参数
    4. **排除异常数据**: 暂时排除流动性不足的货币对
    5. **等待市场稳定**: 在市场异常波动时等待稳定后再分析
    6. **对比其他指标**: 结合其他技术指标验证强度计算
  </Accordion>
</AccordionGroup>

## 篮子交易执行问题

### 篮子交易无法执行

<AccordionGroup>
  <Accordion title="问题描述">
    点击B/S按钮后没有开仓，或只开了部分货币对的仓位
  </Accordion>
  
  <Accordion title="可能原因">
    - EA没有交易权限
    - 账户余额不足
    - 某些货币对不可交易
    - 市场已关闭
    - 点差过大或流动性不足
    - 交易参数设置错误
  </Accordion>
  
  <Accordion title="解决方案">
    1. **检查EA权限**: 确认EA有"允许实时自动交易"权限
    2. **验证账户余额**: 确保有足够的保证金进行篮子交易
    3. **检查市场状态**: 确认相关货币对在交易时间内
    4. **调整交易参数**: 检查手数、止盈止损设置是否合理
    5. **检查货币对状态**: 确认所有货币对都可以交易
    6. **等待市场开盘**: 在市场关闭时等待开盘后再交易
    7. **查看专家日志**: 检查MT5日志中的错误信息
  </Accordion>
</AccordionGroup>

### 自动交易策略不触发

<AccordionGroup>
  <Accordion title="问题描述">
    设置了自动交易策略但达到触发条件时没有执行交易
  </Accordion>
  
  <Accordion title="可能原因">
    - 自动交易开关未启用
    - 触发条件设置错误
    - 强度数据更新延迟
    - 已在当前周期执行过交易
    - 系统时间与服务器时间不同步
  </Accordion>
  
  <Accordion title="解决方案">
    1. **检查开关状态**: 确认自动交易开关已启用
    2. **验证触发条件**: 检查强度阈值和策略设置是否正确
    3. **等待数据更新**: 等待下一个H4周期的数据更新
    4. **检查执行记录**: 确认当前周期是否已执行过交易
    5. **同步系统时间**: 确保系统时间与服务器时间一致
    6. **重置执行状态**: 重启EA重置执行状态
  </Accordion>
</AccordionGroup>

### 篮子交易风险过大

<AccordionGroup>
  <Accordion title="问题描述">
    篮子交易开仓后账户风险过大，保证金使用率过高
  </Accordion>
  
  <Accordion title="可能原因">
    - 交易手数设置过大
    - 同时开启过多篮子
    - 账户杠杆设置不当
    - 相关货币对高度相关
    - 市场波动性增大
  </Accordion>
  
  <Accordion title="解决方案">
    1. **降低交易手数**: 减少每个货币对的交易手数
    2. **限制篮子数量**: 避免同时交易过多货币篮子
    3. **调整杠杆**: 与经纪商协商调整账户杠杆
    4. **分散交易时间**: 不要同时开启所有篮子
    5. **设置止损**: 为所有篮子交易设置合理止损
    6. **监控保证金**: 实时监控保证金使用率
    7. **紧急平仓**: 必要时使用C按钮快速平仓
  </Accordion>
</AccordionGroup>

## 界面操作问题

### 按钮点击无响应

<AccordionGroup>
  <Accordion title="问题描述">
    点击强度卡片或趋势面板的按钮没有反应
  </Accordion>
  
  <Accordion title="可能原因">
    - 鼠标点击位置不准确
    - 按钮区域计算错误
    - 系统响应延迟
    - 事件处理冲突
    - 界面重绘问题
  </Accordion>
  
  <Accordion title="解决方案">
    1. **精确点击**: 确保点击在按钮的有效区域内
    2. **等待响应**: 点击后等待1-2秒再次尝试
    3. **避免连续点击**: 不要连续快速点击同一按钮
    4. **重新加载界面**: 关闭并重新打开面板
    5. **检查系统资源**: 确保CPU使用率不过高
    6. **重启EA**: 重新启动TraderCopilot
  </Accordion>
</AccordionGroup>

### 面板无法拖拽

<AccordionGroup>
  <Accordion title="问题描述">
    无法拖拽移动篮子交易面板或其子面板
  </Accordion>
  
  <Accordion title="可能原因">
    - 拖拽功能被禁用
    - 拖拽区域识别错误
    - CDrag类初始化失败
    - 其他窗口遮挡
    - 系统权限问题
  </Accordion>
  
  <Accordion title="解决方案">
    1. **确认拖拽区域**: 确保在标题栏区域点击拖拽
    2. **检查鼠标操作**: 确认鼠标左键按住并拖拽
    3. **重新初始化**: 关闭并重新打开面板
    4. **检查遮挡**: 确认没有其他窗口遮挡面板
    5. **以管理员运行**: 以管理员权限运行MT5
    6. **重启系统**: 如果问题持续，重启计算机
  </Accordion>
</AccordionGroup>

## 趋势分析问题

### 趋势评分显示异常

<AccordionGroup>
  <Accordion title="问题描述">
    趋势面板中的趋势评分显示为0或异常值
  </Accordion>
  
  <Accordion title="可能原因">
    - CTrendScore类计算错误
    - 技术指标数据不足
    - 货币对历史数据缺失
    - 计算参数设置错误
    - 市场数据质量问题
  </Accordion>
  
  <Accordion title="解决方案">
    1. **检查历史数据**: 确认货币对有足够的历史数据
    2. **等待数据积累**: 新货币对需要等待数据积累
    3. **调整计算参数**: 根据数据情况调整趋势计算参数
    4. **重新计算**: 切换时间周期重新触发计算
    5. **验证数据质量**: 检查价格数据是否有异常
    6. **联系技术支持**: 如果问题持续，联系技术支持
  </Accordion>
</AccordionGroup>

### 历史波幅查询失败

<AccordionGroup>
  <Accordion title="问题描述">
    历史波幅查询无结果或查询过程中出错
  </Accordion>
  
  <Accordion title="可能原因">
    - 查询时间范围过大
    - 选择的货币对无历史数据
    - 网络连接问题
    - 服务器数据不完整
    - 查询参数格式错误
  </Accordion>
  
  <Accordion title="解决方案">
    1. **缩小查询范围**: 选择较短的时间范围进行查询
    2. **更换货币对**: 选择有充足历史数据的货币对
    3. **检查网络**: 确保网络连接稳定
    4. **等待数据同步**: 等待服务器数据同步完成
    5. **验证日期格式**: 确认起始和结束日期格式正确
    6. **分段查询**: 将大范围查询分成多个小段
  </Accordion>
</AccordionGroup>

## 性能问题

### 系统响应缓慢

<AccordionGroup>
  <Accordion title="问题描述">
    篮子交易模块操作响应缓慢，界面卡顿
  </Accordion>
  
  <Accordion title="可能原因">
    - 系统资源不足
    - 同时运行过多EA
    - 位图资源占用过多内存
    - 网络延迟影响数据获取
    - 硬盘读写速度慢
  </Accordion>
  
  <Accordion title="解决方案">
    1. **释放系统资源**: 关闭不必要的程序和EA
    2. **清理位图缓存**: 定期重启面板清理位图资源
    3. **检查网络速度**: 确保网络连接速度足够
    4. **监控内存使用**: 使用任务管理器监控内存
    5. **升级硬件**: 考虑升级内存或使用SSD硬盘
    6. **优化设置**: 减少同时显示的货币对数量
  </Accordion>
</AccordionGroup>

### 内存使用过高

<AccordionGroup>
  <Accordion title="问题描述">
    使用篮子交易模块时，系统内存使用率显著增加
  </Accordion>
  
  <Accordion title="可能原因">
    - 位图资源未及时释放
    - 历史数据缓存过多
    - 内存泄漏问题
    - 图表数据积累过多
  </Accordion>
  
  <Accordion title="解决方案">
    1. **定期重启**: 定期关闭并重新打开面板
    2. **清理缓存**: 清理历史数据和图表缓存
    3. **监控内存**: 定期检查内存使用情况
    4. **限制数据量**: 减少历史查询的数据量
    5. **重启MT5**: 定期重启MT5平台释放内存
    6. **升级内存**: 考虑增加系统内存
  </Accordion>
</AccordionGroup>

## 数据同步问题

### 多面板数据不一致

<AccordionGroup>
  <Accordion title="问题描述">
    主面板、设置面板、分析面板显示的数据不一致
  </Accordion>
  
  <Accordion title="可能原因">
    - 数据更新时机不同步
    - 缓存数据未及时刷新
    - 面板间通信问题
    - 数据源不一致
  </Accordion>
  
  <Accordion title="解决方案">
    1. **强制刷新**: 关闭所有面板重新打开
    2. **同步更新**: 等待所有面板数据更新完成
    3. **检查数据源**: 确认所有面板使用相同数据源
    4. **重启模块**: 重新启动篮子交易模块
    5. **清理缓存**: 清理所有缓存数据重新获取
  </Accordion>
</AccordionGroup>

## 预防措施

### 日常维护

<Tip>
**最佳实践**:
- 定期检查货币强度数据的准确性
- 监控篮子交易的风险水平
- 保持系统时间的准确性
- 定期清理位图和缓存资源
- 备份重要的交易设置
</Tip>

### 风险控制

<Warning>
**重要提醒**:
- 篮子交易具有高风险性，建议先在模拟账户测试
- 合理设置交易手数避免过度杠杆
- 密切监控保证金使用率
- 设置合理的止损保护资金安全
- 避免在重要经济事件时进行大规模篮子交易
</Warning>

联系技术支持时，请准备以下信息：
